# WORKSTREAM 7 — TEST-IMPACT INVENTORY (v0.16)

All paths absolute; line numbers verified against current code. Classification legend: **WB** = will-break (asserts an interface v0.16 removes/changes), **NU** = needs-update (touched code, assertions need adjusting but the test concept survives), **NN** = new-test-needed (no current coverage for the new behavior). Calibration corpora are `tests/calibration/*.jsonl`, driven by `tests/calibration/test_corpus_runner.py`.

A structural caveat that governs everything below: the entire test tree currently constructs `Substrate(...)` / `PredicateTranslation` / `KBVerifier` against the **scalar** `PredicateMetadata` interface (`meta.kb_property`, `meta.slot_to_qualifier`, `meta.single_valued`, `meta.subject_entity_types`/`object_entity_types`). Every `MockTransport.extract_with_tool` in the tree returns the **flat scalar dict** (`{"kb_property": "P39", "slot_to_qualifier": None, "single_valued": 0, ...}`). DATA-MODEL decision says legacy scalar columns are retained and read-synthesized into a single `PredicateBinding`, and `PredicateMetadata` keeps predicate-level fields. **If the read-synthesis path preserves `meta.kb_property` etc. as derived read-only accessors, the vast majority of these tests survive as NU rather than WB.** The inventory flags which tests assert the scalar interface in a way that hard-breaks vs. which merely need their mock transports/fixtures extended to emit `bindings`.

## 4. CALIBRATION-CORPUS DISCIPLINE (do not re-pin to hide regressions)

Single rule for TA-CAL: **a corpus expectation may change only when the underlying behavior changed for a stated architectural reason in this v0.16 build — never to make a red bar green.** Concretely:

- **WS2 derivation flips (abstain→verified):** allowed *only* when premise-forward search genuinely grounds the chain — require the `_run_derivation` trace-entity check (runner L740-753 pattern) to confirm the intended entity/path is in the trace, not just that the verdict string matches.
- **WS4 hard_claim inversions (absent→present-but-abstained):** allowed because verify-every-claim is the explicit decision; the `extraction_corpus` `hard_claim` cases' `expected_subjects_not_in_output` (L275) must be rewritten to `expected_abstention_reason`, not deleted.
- **WS5 intervention dual-designation cases:** *add* `*_given_assertion` input rows with `conditional` expectations to `intervention_corpus.jsonl`; do not retro-fit the existing base-verdict rows.
- **`THRESHOLDS` dict (runner L45-57) is the single source of truth** and is cross-checked by `tests/unit/test_runbook_thresholds.py` against the runbook. **Lowering any threshold to pass a regressed corpus is forbidden** — that doc-test will catch a unilateral threshold edit, but TA-CAL must also not lower the runbook table in lockstep to sneak it through. `kb_mapping_corpus` (multi-binding) and `predicate_metadata_corpus` (binding-level entity types) are the two most at risk of a soundness-hiding re-pin: their "expected" must pin the *evidence-arbitrated winning binding*, not "any binding in the ranked set," or a wrong-primary-property regression passes silently.
