Tier U session-local. User assertions tagged to one conversation via the marker check ("let's say for this conversation…"). is_session_local=1 with exactly one entry in session_ids. The walker prefers these over cross-session matches in the active session.

Tier U cross-session. User assertions that persist across all conversations. is_session_local=0. The session_ids list accumulates which sessions reaffirmed each fact and drives the Beta-posterior reinforcement (one increment per new session).

Model + verifier rows. Facts the assistant asserted in its drafts (asserted_by="model") plus verifier-corrected values (asserted_by="python_verifier"). The dual-write audit trail of model claims that triggered a Layer 5 intervention.

Tier W (world cache). Verifier verdicts on world facts cached with TTL by stability_class. v0.14.1 policy: only actionable verdicts (verified / contradicted) are persisted; the four "we don't know" statuses (retrieval_inconclusive, retrieval_failed, unverifiable_pending_implementation, unverifiable_in_principle) skip the cache so the next attempt can re-try with a different query path. Layer 4's walker consults Tier W after Tier U misses; warm-cache hits short-circuit fresh dispatch.

Layer 2's classification cache. One row per (pattern, predicate) pair the LLM router has classified. After warm-up, lookups short-circuit; counts only change via operator-driven re-judgment (substrate operator endpoints).
